2026 年度資訊部(AP)工作目標

  • 1 系統翻新
    • 1.1 重構跨年官方網站 (含後台)
      依據 SDD 進行重構，改善 10 年舊架構。導入 AI 協助前後端架構優化，提升維護性與開發效率。
      弘立、晨定
    • 1.2 D2K薪資系統重構Web (含後台)
      目前新泉/鼎固/新和/僑嶺薪資系統後台為D2K開發，
      前台報表、電子薪資單為Java/Jsp 介面，
      預計統一翻新為 asp.net 介面，資料庫則為Postgresql DB.
      弘立、晨定
  • 2 系統維運及優化
    • 2.1 主機升級 2025 Server
      針對 192.168.10.2 (新泉/鼎固/薪資)、172.25.202.101 (資網管) 及 EIP 測試機進行升級。
      建宏、晨定
    • 2.2 Datadog 間歇啟用
      執行報價、導入、追蹤、暫停、重啟之循環管理 (Loop)。
      奕均、建宏
  • 3 資訊風險管理
    • 3.1 舊資料管理
      舊檔案 (FTP/Table) 自動清理並移出主機。
      晨定、軒豪
    • 3.2 機敏資料隔離
      Server 端網站改用環境變數抓取 ConnectString 或 API Key，避免明碼外漏。
      建宏
    • 3.3 TLS 憑證展延自動化
      因應憑證效期逐年遞減 (2026年剩200天)，建立自動展延機制。
      晨定
    • 3.4 敏感欄位加密
      針對密碼及 Log 記錄中的敏感欄位進行加密處理。
      建宏
  • 4 EIP 功能優化
    • 4.1 首頁增加動態功能
      列出最近上傳的檔案與公告，避免漏讀未發 Mail 的更新資訊。
      晨定
    • 4.2 通訊資料更新提醒
      每隔 12 個月或職務異動時，系統自動詢問是否修改分機資料。
      晨定
    • 4.3 LINE Rich Menu 管理頁
      區分主管版/User版 Menu，並提供後台介面進行版次管理。
      軒豪
    • 4.4 差勤表單調整後台
      開放請假、加班單之刪除與修改功能 (含 Log 記錄)，減少行政與資訊來回溝通時間。
      軒豪
    • 4.5 SQL 語法全面去除 Schema
      建立 Public Synonyms，移除 SQL 中如 EUEIP.ORG... 的前綴，確保測試/正式環境切換彈性。
      建宏
    • 4.6 請假:零散補休搭配私休使用
      評估補休+私休組合假可行性，處理簽核流程與時數組合問題。
      弘立、軒豪
    • 4.7 EIP 速度過慢程式調效
      歷史查詢資料分流；針對 Datadog 偵測出的慢速程式 (如 abshistory.aspx, orderlist.aspx) 進行優化。
      晨定、軒豪
  • 5 AI 應用擴展
    • 5.1 AI 自動產出系統文件
      研究透過 GitHub Copilot CLI 或 Workspace 自動建置系統 Markdown 文件。
      奕均
    • 5.2 EIP 智慧上傳
      透過 AI 及 OCR 技術，上傳檔案時自動辨識並填入對應欄位。
      建宏
    • 5.3 AI 判讀程式異動說明
      GitHub 週報匯整；EIP 新增功能自動 Insert 至版本記錄，待工程師確認後開放。
      建宏
    • 5.4 助理機器人架構翻新
      採用 MCP (Model Context Protocol) 協議直連本地資源，取代傳統 Lambda API 複雜驗證流程。
      建宏
    • 5.5 外部通訊錄 AI 解析
      每日偵測更新並透過 AI 清洗結構化資料；支援查詢檔名、姓名、電話並引導下載。
      建宏、晨定
    • 5.6 實驗 AI 偵測程式修復
      建立 SonarQube SOP，架設 MCP Server 供 GitHub Copilot 直接抓取 Issue 進行修復。
      建宏
  • 6 自動化整合交付 (CI/CD)
    • 6.1 自動編譯自動佈署
      實作 Windows Server 部署方案 (GitLab Runner 本機建置 或 SSH 遠端部署)。
      軒豪
    • 6.2 程式碼風格檢查
      導入 .editorconfig 與 CI 檢查 (全專案/變更區)，可自動修正或僅推播 Slack 通知。
      軒豪
  • 7 技術研習類
    • 7.1 混合型資料庫 SeekDB 測試
      整合 GIS Query 進行概念測試 (如：查詢方圓 2 公里內的簽約廠商)。
      奕均
    • 7.2 ETL 工具測試 (Apache Hop)
      整合 Apache Hop 進行 ETL 工具測試及應用 (與 #16 整合進行)。
      奕均
    • 7.3 MongoDB NoSQL 研習
      NoSQL 資料庫相關技術研習。
      弘立
    • 7.4 性能驗證工具測試
      使用 JMeter 進行基準、負載、壓力及穩定性測試，監測 TPS, Response Time 等指標。
      奕均
  • 8 支援類
    • 8.1 義享樂業務支援
      配合業務需求提供相關支援。
      弘立
